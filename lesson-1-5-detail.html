📖 术语详解

A
A 开头
API Application Programming Interface
网络
应用程序接口，不同软件系统之间交换数据的标准方式。
详细解释

API 就像是餐厅的服务员。你（客户端）不需要直接进入厨房（服务器），而是通过服务员点餐和取餐。API 定义了请求和响应的格式，让前后端能够顺畅协作。

产品经理关注点

• 了解需要调用哪些接口来实现功能
• 确认接口的输入参数和返回数据格式
• 关注接口的性能（响应时间）

示例
获取用户信息的 API：GET /api/users/123 → 返回用户昵称、头像、手机号等数据
相关术语： REST HTTP JSON
Ajax Asynchronous JavaScript and XML
前端
异步数据请求技术，实现页面不刷新就能获取数据。
详细解释

Ajax 让网页可以在后台悄悄向服务器请求数据，然后局部更新页面内容，而不需要整个页面刷新。这就是为什么你在刷信息流时，新内容能流畅地出现。

应用场景
下拉刷新、加载更多、实时搜索建议、表单自动保存
B
B 开头
后端 / Backend Server-side
后端
运行在服务器端的程序，负责业务逻辑、数据处理和接口提供。
详细解释

后端就像餐厅的厨房，用户看不到，但负责处理所有"核心业务"：验证用户身份、计算订单金额、管理库存、处理支付等。后端代码运行在云服务器上，24小时不停工作。

常用技术栈

Java、Python、Node.js、Go、PHP 等后端编程语言

相关术语： 前端 API 数据库
浏览器 / Browser
前端
用于访问网页的应用程序，如 Chrome、Safari、Edge。
产品经理关注点

• 兼容性：不同浏览器显示效果可能有差异
• 开发者工具：F12 键可打开，用于调试和查看网络请求
• 缓存机制：可能导致用户看不到最新版本

C
C 开头
缓存 / Cache
概念
临时存储常用数据，加快访问速度的技术。
详细解释

缓存就像是把常用的东西放在手边。浏览器缓存图片和脚本文件，避免每次打开都重新下载；服务端缓存热点数据，减少数据库查询。用空间换时间，提升用户体验。

注意事项

缓存可能导致数据不一致。比如用户更新了头像，但还看到旧头像（缓存未过期）。产品经理需要考虑缓存更新策略。

缓存层级
浏览器缓存 → CDN 缓存 → 服务器缓存 → 数据库
客户端 / Client
概念
用户直接使用的应用程序，如 App、网页、小程序。

客户端包括：iOS App、Android App、网页（Web）、微信小程序、桌面应用等。客户端负责展示界面和收集用户操作，通过网络与服务端通信。

Cookie
网络
存储在浏览器中的小型文本文件，用于保存用户状态。
常见用途

• 保持登录状态（记住你是谁）
• 记录用户偏好（如语言选择）
• 追踪用户行为（用于数据分析）

隐私提示
欧盟 GDPR 法规要求网站必须提示用户并获得同意才能使用 Cookie
CSS Cascading Style Sheets
前端
层叠样式表，控制网页的视觉样式（颜色、布局、字体等）。

CSS 负责"美化"网页。HTML 定义结构（这是标题，那是按钮），CSS 定义样式（标题用多大字号、按钮是什么颜色）。产品经理与设计师沟通时，需要关注 CSS 是否能实现设计稿的效果。

D
D 开头
数据库 / Database
数据库
结构化存储和管理数据的系统，相当于数字化的"档案室"。
常见类型

关系型数据库（如 MySQL、PostgreSQL）：用表格存储数据，适合复杂查询
非关系型数据库（如 MongoDB、Redis）：灵活的数据结构，适合高并发场景

产品经理关注点

• 数据表设计是否合理
• 查询性能是否满足需求
• 数据备份和恢复机制

类比
数据库 = 档案室 | 数据表 = 文件柜 | 记录 = 档案袋 | 字段 = 档案中的各个信息项
DOM Document Object Model
前端
文档对象模型，浏览器将 HTML 页面解析成的可操作结构。

DOM 是网页的"骨架图"。浏览器把 HTML 标签转换成树状结构，JavaScript 可以通过 DOM 来增删改页面元素。比如点击按钮弹出提示，就是通过修改 DOM 实现的。

F
F 开头
前端 / Frontend Client-side
前端
运行在用户设备上的程序，负责界面展示和用户交互。
核心技术

HTML：页面结构 | CSS：视觉样式 | JavaScript：交互逻辑

现代前端框架

React、Vue、Angular 等框架让开发复杂应用更高效

职责划分
前端负责"看得见"的部分：布局、动画、表单验证、页面跳转
框架 / Framework
概念
预先定义好的代码结构和工具集合，加速应用开发。
常见框架

前端：React、Vue、Angular
后端：Spring Boot(Java)、Django(Python)、Express(Node.js)
移动端：Flutter、React Native

类比
框架 = 房屋装修模板 | 提供基础结构，开发者只需填充内容
G
G 开头
Git
概念
分布式版本控制系统，用于管理代码的变更历史。
核心概念

Commit：提交代码变更
Branch：分支，并行开发不同功能
Merge：合并分支代码
Pull Request：代码审查请求

产品经理关注点

• 了解开发分支管理策略（如 Git Flow）
• 理解版本发布的代码冻结机制

H
H 开头
HTTP HyperText Transfer Protocol
网络
超文本传输协议，互联网数据交换的基础协议。
HTTP 方法

GET：获取数据 | POST：提交数据 | PUT：更新数据 | DELETE：删除数据

状态码

200(成功)、400(请求错误)、401(未授权)、404(未找到)、500(服务器错误)

HTTPS
HTTP 的安全版本，数据加密传输，防止窃听和篡改
HTML HyperText Markup Language
前端
超文本标记语言，定义网页的内容结构。

HTML 使用标签（如 <div>、<p>、<img>）来组织内容。产品经理不需要写 HTML，但理解它有助于与前端沟通结构需求，也有助于理解技术可行性。

I
I 开头
接口 / Interface
概念
系统之间交互的约定规范，定义了如何请求和返回数据。

接口是前后端协作的"契约"。前端按接口规范发送请求，后端按规范返回数据。接口文档通常包含：接口地址、请求方法、参数说明、响应示例等。

相关术语： API REST
J
J 开头
JavaScript / JS
前端
网页的编程语言，实现动态交互和复杂逻辑。

JavaScript 是让网页"动起来"的魔法。表单验证、页面动画、数据请求、响应用户操作，都靠 JavaScript 实现。现代 JavaScript 也能用于后端开发（Node.js）。

与 Java 的区别
JavaScript 和 Java 是两种不同的语言，只是名字相似。就像"雷锋"和"雷峰塔"的关系。
JSON JavaScript Object Notation
网络
轻量级数据交换格式，易于人阅读和机器解析。

JSON 是 API 接口传输数据的标准格式。它的结构是键值对（key-value），类似 JavaScript 对象。几乎所有编程语言都能解析 JSON。

JSON 示例
{ "name": "张三", "age": 25, "isVip": true }
L
L 开头
LocalStorage
前端
浏览器本地存储，可长期保存数据（除非用户主动清除）。

LocalStorage 用于在浏览器中长期存储数据，如用户偏好设置、草稿内容等。与 Cookie 不同，它不会随请求发送到服务器，存储容量也更大（通常 5MB）。

相关术语： Cookie SessionStorage
M
M 开头
Mock
概念
模拟数据或接口，用于开发和测试阶段。

Mock 数据就像是"假数据"。在真实接口还没准备好时，前端可以用 Mock 数据继续开发页面。产品经理也可以基于 Mock 数据验证交互流程，提前发现问题。

N
N 开头
Node.js
后端
让 JavaScript 可以在服务器端运行的运行环境。

传统上 JavaScript 只能在浏览器运行，Node.js 打破了这一限制。现在可以用 JavaScript 写后端程序，实现"全栈开发"（同一人负责前后端）。

O
O 开头
OAuth Open Authorization
概念
开放授权协议，允许第三方应用访问用户资源而无需密码。

OAuth 就是"微信登录"、"QQ登录"背后的技术。你不需要把微信密码给第三方App，只需授权即可。OAuth 保护用户账号安全，同时简化注册流程。

应用场景
使用微信/微博/QQ账号登录其他App、授权第三方应用访问你的相册
P
P 开头
分页 / Pagination
数据库
将大量数据分成多页显示的技术。

想象一本1000页的书，你不会一次性全部摊开，而是一页页看。分页就是一样的道理：列表数据太多时，只显示前20条，用户点击"加载更多"或"下一页"再获取后续数据。

分页方式

传统分页：上一页/下一页按钮
无限滚动：滚动到底自动加载（如朋友圈）
加载更多：点击按钮加载下一批

PRD Product Requirements Document
概念
产品需求文档，描述产品功能、逻辑和交互的详细说明。

PRD 是产品经理的"施工图纸"。开发团队根据 PRD 理解要做什么、怎么做。一份好的 PRD 应包含：功能描述、业务流程、页面原型、数据规则、异常处理等。

Q
Q 开头
查询 / Query
数据库
从数据库中检索数据的操作。

查询是最常用的数据库操作。用户搜索商品、查看订单列表、筛选用户，都是查询。复杂的查询可能影响性能，产品经理在设计筛选条件时要考虑技术实现成本。

R
R 开头
REST Representational State Transfer
网络
一种设计 API 的架构风格，使用 HTTP 方法操作资源。
RESTful 设计原则

• 使用名词表示资源（/users 而非 /getUsers）
• 用 HTTP 方法表示操作（GET 获取、POST 创建、PUT 更新、DELETE 删除）
• 状态码表示结果（200 成功、404 不存在）

RESTful API 示例
GET /users（获取用户列表）| POST /users（创建用户）| GET /users/123（获取ID为123的用户）
响应式 / Responsive
前端
网页根据设备屏幕大小自动调整布局的技术。

响应式设计让同一个网页在手机、平板、电脑上都能良好展示。产品经理需要考虑不同设备的体验差异，在 PRD 中明确各端的布局要求。

S
S 开头
SQL Structured Query Language
数据库
结构化查询语言，用于操作关系型数据库的标准语言。
基本操作

SELECT：查询数据
INSERT：插入数据
UPDATE：更新数据
DELETE：删除数据

SQL 示例
SELECT * FROM users WHERE age > 18 ORDER BY create_time DESC
服务器 / Server
概念
提供计算服务的计算机或程序，响应客户端的请求。

服务器可以是物理机器（放在机房里的电脑），也可以是云上的虚拟资源。服务器24小时运行，处理来自全球的请求。常见的服务器类型：Web服务器、数据库服务器、文件服务器等。

Session
网络
会话，服务器端保存的用户状态信息。

Session 用于记录用户在一段时间内的状态，如登录信息、购物车内容。与 Cookie 不同，Session 数据存储在服务器端，安全性更高。Session 有过期时间，超时后用户需要重新登录。

SDK Software Development Kit
概念
软件开发工具包，帮助开发者快速接入某项服务。

SDK 就像是"预制菜包"。想接入微信支付？用微信 SDK。想加入地图功能？用高德 SDK。SDK 封装了复杂的技术细节，开发者只需简单调用即可。

T
T 开头
Token
网络
令牌，用于身份验证的加密字符串。

用户登录成功后，服务器生成一个 Token 返回给客户端。之后客户端每次请求都带上这个 Token，服务器通过验证 Token 确认用户身份。Token 比传统的 Session 更适合移动端和分布式系统。

JWT
JSON Web Token，一种流行的 Token 格式，包含用户信息和签名
测试 / Test
概念
验证软件功能正确性的过程。
常见测试类型

单元测试：测试最小代码单元
集成测试：测试模块之间的协作
功能测试：验证业务功能
性能测试：测试系统承载能力
用户测试：真实用户验证体验

U
U 开头
URL Uniform Resource Locator
网络
统一资源定位符，网页或资源的地址。

URL 就是浏览器地址栏里的那串字符，如 https://www.example.com/products/123。它包含协议（https）、域名（www.example.com）和路径（/products/123）。

URL 结构
https://api.example.com:8080/v1/users?id=123#profile
协议://域名:端口/路径?查询参数#锚点
UX / UI
概念
用户体验 / 用户界面，产品设计的核心概念。

UI（User Interface）：用户看到的界面，包括颜色、字体、按钮样式等视觉元素。
UX（User Experience）：用户使用产品的整体体验，包括易用性、效率、情感感受。

类比
UI 是餐厅的装修和摆盘 | UX 是用餐的整体感受（服务、味道、氛围）
V
V 开头
版本控制 / Version Control
运维
管理软件代码不同版本变更的系统。

版本控制让团队可以多人协作开发，记录每次代码修改，出问题可以回滚到之前的版本。Git 是目前最流行的版本控制系统。

W
W 开头
Web
前端
万维网，基于互联网的文档系统，通过浏览器访问。

Web 包括网站（Website）和网页应用（Web App）。Web 应用不需要安装，通过浏览器即可使用，跨平台性好，但功能受浏览器能力限制。

相关术语： 浏览器 HTTP
