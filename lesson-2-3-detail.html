SQL 基础与数据查询

掌握 SQL 的核心语法，学会编写高效的数据查询语句。从基础 SELECT 到复杂的 JOIN 和子查询， 配合产品经理常用的数据分析场景，让你能够自主验证产品数据、排查业务问题。

📝 SQL 基础语法

SQL（Structured Query Language）是操作关系型数据库的标准语言。掌握 SQL 基础语法， 可以帮助产品经理快速查询数据、验证产品逻辑、分析业务指标。

💡 产品经理学 SQL 的核心价值

  • 数据验证：新功能上线后，快速验证数据是否按预期写入
  • 问题排查：用户反馈问题时，查询相关订单/操作记录
  • 业务分析：自主分析数据，不用每次都找开发同学
  • 需求评估：了解现有数据结构，更准确评估新需求

SQL 语句基本结构

SQL 基础查询语句结构
SELECT 列名1, 列名2, ... -- 选择要查询的字段
FROM 表名 -- 从哪个表查询
WHERE 条件 -- 筛选条件（可选）
GROUP BY 分组字段 -- 分组统计（可选）
HAVING 分组条件 -- 分组后筛选（可选）
ORDER BY 排序字段 [ASC|DESC] -- 排序（可选）
LIMIT 数量; -- 限制返回条数（可选）

常用 SQL 关键字

关键字 用途 示例
SELECT 选择要查询的字段，* 表示所有字段 SELECT * FROM users
WHERE 设置查询条件，筛选数据 WHERE status = 1
AND/OR 组合多个条件 WHERE a = 1 AND b = 2
ORDER BY 排序，ASC 升序，DESC 降序 ORDER BY created_at DESC
LIMIT 限制返回结果数量 LIMIT 10
DISTINCT 去重，返回唯一值 SELECT DISTINCT city

🔍 基础查询实战

通过实际案例学习 SQL 查询。假设我们有一个电商订单系统，包含用户表 users 和订单表 orders。

📌 示例 1：查询所有用户

获取用户表中的所有用户信息

SELECT * FROM users;

-- 只查询指定字段
SELECT user_id, username, email FROM users;

📌 示例 2：带条件的查询

查询状态为正常的用户

SELECT * FROM users
WHERE status = 1;

-- 多条件查询
SELECT * FROM users
WHERE status = 1 AND created_at >= '2024-01-01';

-- OR 条件查询
SELECT * FROM users
WHERE city = '北京' OR city = '上海';

📌 示例 3：排序和分页

查询最新注册的10个用户

-- 按注册时间倒序，取前10条
SELECT user_id, username, created_at
FROM users
ORDER BY created_at DESC
LIMIT 10;

-- 分页查询：第2页，每页20条
SELECT * FROM users
LIMIT 20 OFFSET 20;

⚠️ 常用 WHERE 条件写法

  • = 等于、!= 或 <> 不等于
  • > 大于、< 小于、>= 大于等于、<= 小于等于
  • BETWEEN 范围查询，如 age BETWEEN 18 AND 30
  • IN 多值匹配，如 city IN ('北京', '上海', '广州')
  • LIKE 模糊查询，如 username LIKE '%张%'
  • IS NULL / IS NOT NULL 空值判断

📊 数据统计与聚合

聚合函数是数据分析的利器，可以快速统计业务指标。产品经理最常用的就是这类查询。

🔢

COUNT()

统计记录数量。
如：统计总用户数、订单数

➕

SUM()

求和计算。
如：计算总销售额、总积分

📈

AVG()

平均值计算。
如：平均客单价、平均年龄

⚡

MAX/MIN

最大最小值。
如：最高订单金额、最早注册时间

聚合查询实战

📌 示例 4：统计用户总数

-- 统计总用户数
SELECT COUNT(*) AS total_users FROM users;

-- 统计今日新增用户
SELECT COUNT(*) AS new_users
FROM users
WHERE DATE(created_at) = CURDATE();

📌 示例 5：分组统计（GROUP BY）

统计各状态订单数量和金额

SELECT
  status,
  COUNT(*) AS order_count,
  SUM(total_amount) AS total_revenue,
  AVG(total_amount) AS avg_amount
FROM orders
WHERE created_at >= '2024-01-01'
GROUP BY status;

-- 带条件的分组统计（HAVING）
SELECT user_id, COUNT(*) AS order_count
FROM orders
GROUP BY user_id
HAVING order_count >= 5;
-- 查询下单5次以上的用户

📌 示例 6：多维度数据分析

按日期统计每日订单量和销售额

SELECT
  DATE(created_at) AS order_date,
  COUNT(*) AS order_count,
  SUM(total_amount) AS daily_revenue
FROM orders
WHERE created_at >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY DATE(created_at)
ORDER BY order_date DESC;

🔗 表关联查询（JOIN）

实际业务中，数据往往分散在多个表中。JOIN 用于关联查询多个表的数据，是 SQL 中最重要的技能之一。

🔗 JOIN 类型说明

  • INNER JOIN：只返回两个表匹配的数据（最常用）
  • LEFT JOIN：返回左表所有数据，右表不匹配填 NULL
  • RIGHT JOIN：返回右表所有数据，左表不匹配填 NULL
  • FULL JOIN：返回两个表的所有数据（MySQL 不支持）

📌 示例 7：INNER JOIN 基础关联

查询订单及对应的用户信息

SELECT
  o.order_id,
  o.total_amount,
  u.username,
  u.phone
FROM orders o
INNER JOIN users u ON o.user_id = u.user_id
WHERE o.status = 'paid'
ORDER BY o.created_at DESC
LIMIT 20;

📌 示例 8：LEFT JOIN 保留左表数据

查询所有用户及其订单数（包括未下单的用户）

SELECT
  u.user_id,
  u.username,
  COUNT(o.order_id) AS order_count
FROM users u
LEFT JOIN orders o ON u.user_id = o.user_id
GROUP BY u.user_id, u.username;
-- 未下单的用户会显示 order_count = 0

📌 示例 9：多表关联

查询订单详情，包含用户信息、商品信息

SELECT
  o.order_id,
  u.username,
  p.product_name,
  oi.quantity,
  oi.subtotal
FROM orders o
JOIN users u ON o.user_id = u.user_id
JOIN order_items oi ON o.order_id = oi.order_id
JOIN products p ON oi.product_id = p.product_id
WHERE o.order_id = 'ORD2024001';

💼 产品经理实战场景

掌握这些 SQL 查询技能，你可以独立完成日常的数据验证和分析工作。

场景一：验证新功能数据

🎯 检查优惠券功能是否正常

-- 查看今天使用优惠券的订单
SELECT
  order_id,
  coupon_code,
  discount_amount,
  total_amount - discount_amount AS final_amount
FROM orders
WHERE coupon_code IS NOT NULL
  AND DATE(created_at) = CURDATE()
LIMIT 20;

场景二：用户行为分析

🎯 分析用户留存情况

-- 查询最近30天每天的新增用户和活跃用户
SELECT
  DATE(created_at) AS dt,
  COUNT(*) AS new_users,
  COUNT(DISTINCT CASE WHEN last_login >= DATE_SUB(CURDATE(), INTERVAL 1 DAY) THEN user_id END) AS active_users
FROM users
WHERE created_at >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
GROUP BY DATE(created_at)
ORDER BY dt DESC;

场景三：业务指标监控

🎯 核心指标日报

-- 今日核心指标统计
SELECT
  '今日' AS period,
  COUNT(*) AS order_count,
  SUM(total_amount) AS gmv,
  COUNT(DISTINCT user_id) AS paid_users,
  SUM(total_amount) / COUNT(*) AS avg_order_value
FROM orders
WHERE DATE(created_at) = CURDATE()
  AND status = 'paid';
