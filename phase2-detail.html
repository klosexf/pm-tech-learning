🎯 核心概念预览

主键与外键

主键(PK)唯一标识一条记录，外键(FK)建立表与表之间的关联关系

索引 Index

加速数据查询的数据结构，如手机号、订单号等字段常需建立索引

事务 Transaction

保证一组数据库操作要么全部成功，要么全部失败，确保数据一致性

范式 Normalization

数据表设计的规范化原则，减少数据冗余，避免更新异常

SELECT u.username, o.order_id, o.total_amount
FROM users u
JOIN orders o ON u.user_id = o.user_id
WHERE o.status = 'paid'
ORDER BY o.created_at DESC;
