通过真实电商订单案例，学习如何设计完整的数据模型、统计分析维度和数据展示方案。掌握从需求分析到数据落地的全流程实战技能。

📋 项目背景

  • 业务场景：B2C 电商平台，支持商品购买、订单管理、物流跟踪
  • 用户规模：日活 50万，日均订单 10万+
  • 核心功能：购物车、下单、支付、发货、退款
  • 数据需求：订单查询、销售统计、用户分析、运营报表

订单表（t_order）设计

CREATE TABLE `t_order` ( `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '订单ID', `order_no` VARCHAR(32) NOT NULL UNIQUE COMMENT '订单编号', `user_id` BIGINT NOT NULL COMMENT '用户ID', `total_amount` DECIMAL(10,2) NOT NULL COMMENT '订单总金额', `discount_amount` DECIMAL(10,2) DEFAULT 0 COMMENT '优惠金额', `pay_amount` DECIMAL(10,2) NOT NULL COMMENT '实付金额', `freight_amount` DECIMAL(10,2) DEFAULT 0 COMMENT '运费', `status` TINYINT NOT NULL DEFAULT 0 COMMENT '订单状态：0待付款 1已付款 2已发货 3已完成 4已取消', `pay_type` TINYINT COMMENT '支付方式：1微信 2支付宝 3银行卡', `pay_time` DATETIME COMMENT '支付时间', `receiver_name` VARCHAR(50) COMMENT '收货人姓名', `receiver_mobile` CHAR(11) COMMENT '收货人电话', `receiver_address` VARCHAR(200) COMMENT '收货地址', `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间', `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间', INDEX `idx_user_id` (`user_id`), INDEX `idx_status` (`status`), INDEX `idx_created_at` (`created_at`) ) COMMENT '订单表';

订单商品表（t_order_item）设计

CREATE TABLE `t_order_item` ( `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT 'ID', `order_id` BIGINT NOT NULL COMMENT '订单ID', `product_id` BIGINT NOT NULL COMMENT '商品ID', `product_name` VARCHAR(128) NOT NULL COMMENT '商品名称（快照）', `product_image` VARCHAR(256) COMMENT '商品图片', `product_spec` VARCHAR(100) COMMENT '商品规格', `unit_price` DECIMAL(10,2) NOT NULL COMMENT '单价', `quantity` INT NOT NULL COMMENT '数量', `subtotal` DECIMAL(10,2) NOT NULL COMMENT '小计金额', `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间', INDEX `idx_order_id` (`order_id`), INDEX `idx_product_id` (`product_id`) ) COMMENT '订单商品表';
💡 设计要点

为什么订单商品表要存储商品快照？ 因为商品价格可能变动，如果不保存下单时的价格，后续查看订单就会出现金额不匹配的问题。

📈 统计维度设计

电商数据分析通常从以下几个维度展开：

时间维度

  • 实时数据：当前在线、今日累计、实时销售额
  • 日维度：每日订单量、每日销售额、每日客单价
  • 周维度：周环比、周内分布（分析周末效应）
  • 月维度：月度趋势、月度目标完成率
  • 自定义：活动期间的专项统计分析

商品维度

  • 单品分析：销量排行、销售额排行、转化率
  • 品类分析：品类销售占比、品类增长趋势
  • 品牌分析：品牌销售分布、品牌复购率

用户维度

  • 新老用户：新客订单占比、老客复购率
  • 用户等级：会员等级分布、高价值用户贡献
  • 地域分布：省份/城市销售排行、地域渗透率

📌 产品经理实践建议

  • 先定义核心业务指标（北极星指标），再拆解细分维度
  • 埋点设计要在需求阶段就考虑，避免后期补数据
  • 统计口径要统一，比如「订单金额」是否包含运费、优惠
  • 提供下钻能力，从汇总数据可以查看明细

🎓 章节总结

📌 核心收获

  • 数据表设计：订单主表 + 订单商品表，注意商品快照存储
  • 金额计算：总金额 = 商品金额 + 运费 - 优惠金额
  • 状态流转：待付款 → 已付款 → 已发货 → 已完成
  • 统计维度：时间、商品、用户三大分析维度
  • 列表设计：查询效率、信息密度、视觉层级三者平衡
